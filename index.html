The Basics

Data Types

A.K.A variables

Python works with both numeric and character data

Integer

Is latin for whole number; cannot represent a fractional number


                          a = 5

                          b = -1000000

                          [in]: a / 2

                          [out]: 2

Good for counting, ranking and categorising

Floating point

Approximates a real number to a fixed number of significant digits


                          c = 4.523435

                          d = -1.3e10

                          [in]: c / 10.0

                          [out]: 45.23435

Good for continuous values, measurements, parameters

Boolean

Represents a binary outcome: true or false; actually a subclass of integer


                          e = True  #1

                          f = False #0

                          [in]: e == f

                          [out]: False

Good for dummy variables/flags and conditions

Strings

Represents a single character or grouping of letters, defined by surround quotation marks


                          letter = "x"
                          word = "old"
                          name = "Fred"
                          number = "100"

...we're going to come back to this later on though

Your code is built on these data types, and it is important that you know how to use them

Data structures

or containers that can hold many items

Universal attributes

Items are accessed by position or reference key

and

Are characterised by their size or length

Strings are such an example

so let's revisit them right now

Strings...again!

Are actually a list of symbolic characters


                          name = "Fred"
                          name = ["F", "r", "e", "d"]

with each character having a position in the list


                          ["F", "r", "e", "d"]
                          [ 0 ,  1 ,  2 ,  3 ]

the term index is used to denote position and always starts at 0

Python is quite funky in that you can index from left to right


                          ["F", "r", "e", "d"]
                          [ 0 ,  1 ,  2 ,  3 ]

and right to left


                          ["F", "r", "e", "d"]
                          [-4 , -3 , -2 , -1 ]

Lists

Denoted by square brackets

Is dynamic; size can be changed at any time - is not fixed

Tuples

Denoted by circle brackets


                          tuple_ = (1, 3, True, "hello")

Size is fixed and cannot be changed
